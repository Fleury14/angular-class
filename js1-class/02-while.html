While Loops

Now that we've gone over for loops, let's look over another type, the while loop.

A Basic Example

let counter = 0;

while (counter < 10) {

console.log(`The number is ${i});

counter++;

}

One of the advantages to the while loop is that it's a little bit easier to read. We can see that we declare a variable named counter, and set it equal to 0. We then have a block of code that is repeated as long as the value of counter is less than ten. After our console log, we increment our counter, and then run the code again.

This can be read like a normal sentence: "While counter is less than 10, do this." This is one of the main advantages and is why in my opinion it can seem easier to learn. There are two main behavioral difference between while and for:

Difference 1: Block, Interrupted

Unlike a for loop which only checks its condition after the block of code has been repeated, while loops check the condition all the time. This means that in our example, as soon as counter reaches 10, which makes our condition false, it exits out of the loop and ignores any code in the block that is below it. So for example if we added the following to our example:

let counter = 0;

while (counter < 10) {

console.log(`Before increment: The number is ${i});

counter++;

console.log(`After increment: The number is ${i});

}

If we run this, you will not see the second console log for the number 10; as soon as counter hits 10, Javascript says "peace out" and leaves the loop immediately and doesn't care about the second console log.

Difference 2: With great looping comes great responsibility...

This is the big one. In a for loop, we HAVE to declare our incrementor in order for our loop to work at all. In a while loop, it is the coder's responsibility to throw the incrementor in the block. This means if you try to get fancy and do something like throw the incrementor inside a conditional statement that may not always run, the loop may run forever. This is (in my opinion) the biggest reason you don't see while loops quite as much in JavaScript: it is really easy to mess them up.

Fun tip: If you are writing code in an environment that constantly refreshes like CodePen, be sure to disable that behavior before writing a while loop. You've been warned.

Is there a way to make sure the block of code runs at least once?

What a strangely particular question. And the answer is yes. Enter the do..while loop.

Do..While Example

let counter = 0;

do {

console.log(`The number is ${i});

counter++;

}

while (counter < 10)

Using do..while means that the block of code will always run at least once, even if the condtion is false. So if we changed our condition to while (counter > 100). It would still run once, because the condition doesn't begin evaluating until the first block is run.

All of the other behaviors of a while loop are retained here: As soon as the condition is false, the loop terminates regardless of whether or not the block is done. The pitfall is true as well; it is on the coder to make sure the to include the incrementer in such a way that the loop doesn't go on forever.

When do I use for and when do I use while?

In my (limited) experience, I use for loops 99% of the time. While loops may be easier to read, but as your flow becomes more complicated, it is easier to use the for loop to run an entire block of code a set amount of times. There is some usefulness in utilizing the while loops immediate break out of a block, but it's also balanced out by being really easy to break.

As you get more proficient with loops, you may find yourself repeating the same style of loop for the same purpose. This is why some of the more advanced loops came to be, that we will explore in a new lesson.
