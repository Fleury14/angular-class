Advanced Loops

You may notice, particularly when dealing with arrays, that you find yourself writing the same for loop. Fortunately, JavaScript has even more tools, particularly with arrays. I'll post an example and explain the notation.

Example 1: forEach()

The forEach loop allows us to iterate over an entire array. Instead of having to use a variable to increment up to the array length, you are give each value of the array.

const captains = ['Kirk', 'Picard', 'Sisko', 'Janeway', 'Archer', 'Lorca']

captains.forEach( eachCaptain => {

console.log('This is a captain:', eachCaptain);

});

There's a lot to take in here as this may be your first time seeing arrow notation. You will see this style of notation a lot, especially if you go on to Angular where it is used almost exclusively to its counterpart, which would be an anonymous function (we'll get to that later)

With this notation, declaring a value named eachCaptain. The name of this is not important, it can be any name you want as it is simply a variable name. What is important to know is that it's value will contain each value of the array, one at a time. That's why I gave it the name eachCaptain, so it's easier visually.

We then apply our variable which now represents each value and it will run the code inside the block after the phat arrow => once for each item in the array, with our variable contatin each value. If you run this code, you will see a list of captains in the console log.

What I want to get the index?

While one of the benefits of this loop is that we don't need the index, sometimes you will. That can be passed into our block like so:

const captains = ['Kirk', 'Picard', 'Sisko', 'Janeway', 'Archer', 'Lorca']

captains.forEach( (eachCaptain, index) => {

console.log(`Captain ${eachCaptain} has an index of ${index});

});

Now we have the index, AND the value! Two for the price of one!

Example 2: for..of

For..of alows you to iterate over any kind of iterable objects. Most commonly, this is generally used with arrays, and sometimes strings. This will not work with objects though! We can write our previous array like so:

const captains = ['Kirk', 'Picard', 'Sisko', 'Janeway', 'Archer', 'Lorca']

for (let eachCaptain of captains) {

console.log(`Hello, Captain ${eachCaptain}!);

});

With this style of loop, the variable eachCaptain is still representing each iteration of the array. There is a way to grab the index if you like, but it's a little more convoluted. I suggest either using forEach, or reading up on the MDN entry for for..of.

Tip: If you are planning on moving on to Angular, familiarize yourself with for..of. You will see it a lot.

Example 3: for..in

For..in is primarily used to loop over properties of an Object. I kow we haven't gone over objects yet, so at this point, try to be familiar with what it's doing and know that if you ever need to go over an object, this is a great way to do it:

const myInfo = {

firstName: 'J.R.',

lastName: 'Ruggiero',

age: 39

};

for (let property in myInfo) {

console.log( myInfo[property] )

}

// Result: J.R., Ruggiero, 39

This time, instead of the variable we create representing each value in an array, it represents each property in the object. We then console log the value for each of those properties.

Try to avoid using for..in with arrays. While it will technically work, it can have some unexpected side effects. I recommend sticking to for..of or forEach for arrays.
